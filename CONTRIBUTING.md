# コントリビューションガイド

## 文字コードと改行

- テキストはUTF-8（BOMなし）、改行はLFを使用します。

## 命名規則

- 部ディレクトリ・部ファイル: `第N部_部の題名`
- 章ファイル: `第NN章_章の題名.md`（章番号は2桁）
- 画像: `第NN章_内容_連番.拡張子`
- 日本語の区切りにはアンダースコアを使い、空白は使いません。
- Markdown内のリンクは相対パスで記述します。

## 章の構成

`chapter-template.md` を複製し、章タイトル、この章で学ぶこと、前提知識、本文、ハンズオン／演習、まとめ、用語集、参考資料の順に記載します。

未確定事項には `TODO:`、事実確認が必要な箇所には `要確認:` を付けます。機密情報や実在する認証情報はコミットしません。

## 変更とレビューの手順

1. Issueまたは変更目的を確認し、`main` から作業ブランチを作ります。
2. 章単位で変更し、リンクと画像表示を確認します。
3. 誤字、技術的正確性、コマンドの安全性、出典をセルフレビューします。
4. Pull Requestに変更概要、対象読者、確認方法、関連Issueを記載します。
5. レビュアーは、正確性、学習目標との対応、用語と番号の一貫性、リンク、著作権、安全性、HTML/PDF変換可否を確認します。
6. 原則1名以上の承認を得て、指摘を解決してから `main` にマージします。

内容変更は `CHANGELOG.md` の「未リリース」に追記してください。
