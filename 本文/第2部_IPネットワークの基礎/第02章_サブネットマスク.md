# 第02章 サブネットマスク

**― IPアドレスからネットワークの範囲を読み取る ―**

> この章では、IPv4アドレスをネットワーク部分とホスト部分に分けるサブネットマスクと、CIDR表記による範囲の求め方を学びます。

------------------------------------------------------------------------

# 1. この章で学べること

- サブネットとサブネットマスクの役割
- プレフィックス長とCIDR表記
- ネットワークアドレスとブロードキャストアドレス
- 同一ネットワークかどうかの判断
- Linuxでアドレス範囲と経路を確認する方法

# 2. この章の位置付け

第1章ではIPアドレスが届け先を表すことを学びました。本章では、IPアドレスから「同じネットワークにいる相手か」「ルータへ渡す相手か」を判断するための範囲を扱います。

# 3. なぜサブネットマスクが必要になったのか

IPアドレスだけを見ても、どの部分がネットワークを表し、どの部分がその中のインターフェースを表すかは分かりません。すべてを一つの巨大なネットワークにすると、管理やブロードキャストの範囲が大きくなり、組織ごとの分割もできません。

そこでネットワークを適切な大きさに分割する**サブネット（Subnet）**と、IPv4アドレスのネットワーク部分を示す**サブネットマスク（Subnet Mask）**が使われます。

# 4. 技術の概要

IPv4アドレスは、次の二つの部分として解釈されます。

- ネットワーク部分：所属するネットワークを表す
- ホスト部分：そのネットワーク内のインターフェースを表す

例えば `192.0.2.10/24` では、先頭24ビットがネットワーク部分、残り8ビットがホスト部分です。

`/24` のような表記を**プレフィックス長（Prefix Length）**と呼びます。アドレスとプレフィックス長を組み合わせる方式は、**CIDR（Classless Inter-Domain Routing）**表記と呼ばれます。

# 5. 詳しい仕組み

## サブネットマスクとプレフィックス長

サブネットマスクは、ネットワーク部分を2進数の1、ホスト部分を0で表します。

```text
IPアドレス       192.0.2.10
サブネットマスク 255.255.255.0
2進数            11111111.11111111.11111111.00000000
プレフィックス長 /24
```

代表的な対応は次のとおりです。

| プレフィックス長 | サブネットマスク | 総アドレス数 |
|---:|---|---:|
| `/8` | `255.0.0.0` | 16,777,216 |
| `/16` | `255.255.0.0` | 65,536 |
| `/24` | `255.255.255.0` | 256 |
| `/25` | `255.255.255.128` | 128 |
| `/26` | `255.255.255.192` | 64 |
| `/27` | `255.255.255.224` | 32 |
| `/28` | `255.255.255.240` | 16 |

IPv4の総アドレス数は、ホスト部分のビット数を `h` とすると `2^h` で求められます。

## ネットワークアドレス

**ネットワークアドレス（Network Address）**は、ホスト部分をすべて0にしたアドレスで、サブネット自体を表します。

`192.0.2.10/24` のネットワークアドレスは `192.0.2.0` です。

## ブロードキャストアドレス

**ブロードキャストアドレス（Broadcast Address）**は、IPv4でホスト部分をすべて1にしたアドレスです。同じブロードキャストドメインの全端末宛てに使われます。

`192.0.2.10/24` のブロードキャストアドレスは `192.0.2.255` です。IPv6にはIPv4と同じブロードキャストはなく、マルチキャストを利用します。

## 利用可能なホストアドレス

一般的なIPv4サブネットでは、ネットワークアドレスとブロードキャストアドレスを端末へ割り当てません。

```text
192.0.2.0/24
ネットワークアドレス     192.0.2.0
利用可能範囲             192.0.2.1 ～ 192.0.2.254
ブロードキャストアドレス 192.0.2.255
```

ただし、ポイントツーポイントリンクで使う `/31` など例外があります。

## `/26` の計算例

`192.0.2.70/26` を考えます。`/26` は1サブネット64アドレスなので、第4オクテットの境界は0、64、128、192です。

```text
70が含まれる範囲         64 ～ 127
ネットワークアドレス     192.0.2.64
利用可能範囲             192.0.2.65 ～ 192.0.2.126
ブロードキャストアドレス 192.0.2.127
```

# 6. Linuxではどうなるか

```bash
# IPアドレスとプレフィックス長を表示
ip -br address

# カーネルが認識している直結経路を表示
ip route

# ipcalcが導入済みならアドレス範囲を計算
ipcalc 192.0.2.70/26
```

代表的な出力例（必要な部分のみ抜粋）

```text
$ ip -br address
eth0    UP    192.0.2.70/26

$ ip route
192.0.2.64/26 dev eth0 proto kernel scope link src 192.0.2.70

$ ipcalc 192.0.2.70/26
Network:   192.0.2.64/26
HostMin:   192.0.2.65
HostMax:   192.0.2.126
Broadcast: 192.0.2.127
```

確認ポイント

- `192.0.2.70/26` の `/26` がプレフィックス長です。
- `192.0.2.64/26` がLinuxの認識する直結ネットワークです。
- `scope link` は、その経路が同じリンク上へ直接到達できることを示します。
- `ipcalc` の `Network`、`HostMin`、`HostMax`、`Broadcast` で計算結果を確認できます。

`ipcalc` の書式は実装により異なり、標準で導入されていない環境もあります。試験ではツールに頼らず計算できることも必要です。

# 7. 実務ではどう使われるか

## 実務コラム：同じアドレス帯に見えるのに通信できない

端末Aが `192.0.2.10/24`、端末Bが `192.0.2.130/25` の場合、端末AはBを同一ネットワークと判断します。一方、端末Bの `/25` は `192.0.2.128〜255` なので、Aを別ネットワークと判断します。

このようにプレフィックス長が不一致だと、片方向だけARPを試すなど、分かりにくい障害になります。IPアドレスだけでなく、必ずプレフィックス長も比較します。

```bash
ip -br address
ip route get 192.0.2.130
```

代表的な出力例（必要な部分のみ抜粋）

```text
$ ip route get 192.0.2.130
192.0.2.130 dev eth0 src 192.0.2.10
```

確認ポイント

- `via` がなく `dev eth0` と表示される場合、Linuxは宛先を同じリンク上と判断しています。
- 相手側でも同じ確認を行い、双方の判断が一致しているかを確認します。

# 8. FE/APではどう問われるか

IPアドレスとプレフィックス長から、ネットワークアドレス、ブロードキャストアドレス、利用可能ホスト数、同一サブネットかどうかを求める問題が頻出です。2進数とアドレス境界の両方で計算できるようにします。

# 9. まとめ

- サブネットマスクはIPv4アドレスのネットワーク部分を示します。
- `/24` などのプレフィックス長は、先頭から何ビットがネットワーク部分かを表します。
- ネットワークアドレスはホスト部がすべて0、ブロードキャストアドレスはすべて1です。
- 通信障害ではIPアドレスだけでなくプレフィックス長も確認します。

# 10. 理解度チェック

1. `192.0.2.10/24` のネットワークアドレスとブロードキャストアドレスを求めてください。
2. `/26` のサブネットマスクと総アドレス数を答えてください。
3. `192.0.2.150/26` の利用可能なホストアドレス範囲を求めてください。
4. IPv4で通常、総アドレス数から利用可能ホスト数を2減らすのはなぜですか。

# 11. 解答・解説

## 問1

ネットワークアドレスは `192.0.2.0`、ブロードキャストアドレスは `192.0.2.255` です。

## 問2

サブネットマスクは `255.255.255.192` です。ホスト部分が6ビットなので、総アドレス数は `2^6 = 64` です。

## 問3

`/26` の境界は64ずつです。150は128〜191に含まれるため、ネットワークアドレスは `192.0.2.128`、ブロードキャストアドレスは `192.0.2.191`、利用可能範囲は `192.0.2.129〜190` です。

## 問4

一般的なサブネットでは、ホスト部がすべて0のネットワークアドレスと、すべて1のブロードキャストアドレスを端末へ割り当てないためです。

# 12. 実務で考えてみよう

## ケース：60台の端末を収容するサブネット

### 問題

IPv4で60台の端末を一つのサブネットへ収容するには、最低何ビットのホスト部分が必要ですか。また、プレフィックス長はいくつですか。

### 解答例

ホスト部分5ビットでは `2^5 - 2 = 30` 台で不足します。6ビットなら `2^6 - 2 = 62` 台なので収容できます。IPv4は32ビットのため、プレフィックス長は `32 - 6 = /26` です。将来増加やゲートウェイなどの割り当ても考慮し、実務では余裕を持たせます。
