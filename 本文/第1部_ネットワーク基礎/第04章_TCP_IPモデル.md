# 第04章 TCP/IPモデル

**― インターネットを実際に動かす階層構造 ―**

> この章では、インターネットで利用されるTCP/IPモデルの各層と、OSI参照モデルとの関係を学びます。

------------------------------------------------------------------------

# 1. この章で学べること

- TCP/IPモデルが必要になった背景
- 四つの層と代表的なプロトコル
- OSI参照モデルとの対応と違い
- Linuxのプロトコルスタックを確認する方法

# 2. この章の位置付け

OSI参照モデルは通信を説明するものさしです。本章のTCP/IPモデルは、現在のインターネットを支えるプロトコル群を実用的に整理したモデルです。

# 3. なぜTCP/IPモデルが必要になったのか

異なる種類のネットワーク同士を接続し、一部の経路や機器が変わっても通信を続けるには、特定メーカーや伝送方式に依存しない共通プロトコルが必要でした。

**TCP/IP（Transmission Control Protocol / Internet Protocol）**は、TCPとIPだけではなく、インターネット通信に関係するプロトコル群全体を指す名称として使われます。TCP/IPモデルは、それらを役割ごとに整理します。

# 4. TCP/IPモデルの四つの層

| TCP/IPモデル | 主な役割 | 代表例 | OSI参照モデルとの対応目安 |
|---|---|---|---|
| アプリケーション層 | 利用者向け通信機能 | HTTP、DNS、SSH | 第5〜7層 |
| トランスポート層 | アプリケーション間の配送 | TCP、UDP | 第4層 |
| インターネット層 | ネットワーク間の配送 | IP、ICMP | 第3層 |
| ネットワークインターフェース層 | 同一リンク内の伝送 | Ethernet、Wi-Fi | 第1〜2層 |

対応は理解のための目安であり、二つのモデルを完全に同一視することはできません。

# 5. 詳しい仕組み

## アプリケーション層

Web、名前解決、リモートログインなど、アプリケーションが利用する通信規則を扱います。OSIのセッション層やプレゼンテーション層に相当する機能も、多くの場合はアプリケーション側で扱います。

## トランスポート層

一台のコンピュータで動く複数のアプリケーションを区別し、相手のアプリケーションへ届けます。**ポート番号（Port Number）**はアプリケーションを識別する番号です。TCPは順序制御や再送を行い、UDPはそれらを最小限にして軽量に送ります。詳細は第2部で扱います。

## インターネット層

**IP（Internet Protocol）**はIPアドレスを使って異なるネットワーク間にデータを届けます。配送を試みますが、到着、順序、重複しないこと自体は保証しません。必要な信頼性はTCPなど上位層が補います。

## ネットワークインターフェース層

EthernetやWi-Fiなどを使い、直接接続された範囲でフレームを送ります。同じIP通信でも、途中のリンクごとに伝送方式やフレームは変わり得ます。

## 層をまたぐ処理

送信側ではアプリケーションデータにTCP、IP、リンク層の情報が付きます。ルータは主にIPヘッダを見て次へ転送し、受信側は逆順に処理して対象アプリケーションへ渡します。

# 6. Linuxではどうなるか

LinuxカーネルにはTCP/IPの処理を行う**ネットワークスタック（Network Stack）**があります。アプリケーションは**ソケット（Socket）**という通信の窓口を通して利用します。

```bash
# インターフェースとIPアドレス
ip address show

# IPの経路選択
ip route show
ip route get 192.0.2.10

# TCP/UDPソケットとプロセス
ss -tulnp

# ネットワーク統計
ip -s link
nstat
```

`ss -tulnp` は管理権限がないとプロセス情報の一部を表示できません。コマンド結果を共有するときは、社内IPアドレスなどの情報を不用意に公開しないよう注意します。

# 7. 実務ではどう使われるか

## 実務コラム：ポートは待ち受けているのに接続できない

サーバで `ss -lnt` に対象ポートが表示されても、クライアントから接続できるとは限りません。待受アドレスが `127.0.0.1` だけ、IP経路がない、ホストやネットワークのFirewallが遮断している、といった原因があります。

```bash
ss -lntp
ip route get 198.51.100.20
```

アプリケーション、トランスポート、インターネット、リンクの各層を分けて確認します。

# 8. FE/APではどう問われるか

TCP/IP各層とプロトコルの対応、TCPとUDPの特徴、IPの役割、ポート番号の意味が問われます。「TCP/IP」という名前でもTCPとIPだけを指すとは限らない点を理解します。

# 9. まとめ

- TCP/IPモデルはインターネットのプロトコル群を四層で整理します。
- IPはネットワーク間、TCP/UDPはアプリケーション間の配送を担当します。
- Linuxではカーネルのネットワークスタックをソケット経由で利用します。

# 10. 理解度チェック

1. TCP/IPモデルの四つの層を下位から挙げてください。
2. IPが保証しない信頼性をTCPはどのように補いますか。
3. ポート番号は何を識別しますか。

# 11. 解答・解説

## 問1

ネットワークインターフェース層、インターネット層、トランスポート層、アプリケーション層です。

## 問2

TCPはデータの順序を管理し、受信確認を行い、失われたデータを再送します。これによりアプリケーションへ信頼性のあるバイト列を提供します。

## 問3

一台の機器上で通信を利用するアプリケーションやサービスを識別します。IPアドレスが機器への届け先、ポート番号がその中の窓口に相当します。

# 12. 実務で考えてみよう

## ケース：サーバ自身からは接続できるが別端末から接続できない

### 解答例

サービスがループバックアドレスだけで待ち受けていないか、サーバのFirewall、クライアントからの経路、途中のアクセス制御を確認します。`ss -lntp` のLocal Addressと `ip route` を根拠に切り分けます。
