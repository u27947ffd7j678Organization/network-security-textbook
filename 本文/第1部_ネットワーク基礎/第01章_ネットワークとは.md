# 第01章 ネットワークとは

# 1. この章で学べること

-   ネットワークとは何か

# 2. この章の位置付け

ネットワーク技術は、一つひとつの技術を個別に覚えても全体像を理解しにくい分野です。

そのため、この章では個々の技術の詳細には踏み込まず、**ネットワーク全体の地図**を頭の中に作ることを目的とします。

# 3. なぜネットワークが必要になったのか

コンピュータが誕生した当初は、一台のコンピュータだけで処理を行うことが一般的でした。

しかし、

-   離れた場所にあるコンピュータへデータを送りたい
-   プリンタを共有したい
-   複数人で同じファイルを扱いたい

という要求が増え、コンピュータ同士を接続する**ネットワーク**という考え方が生まれました。

最初は建物内だけの小さなネットワークでしたが、大学、企業、研究機関同士が接続され、現在のインターネットへ発展しました。

つまりネットワークは、

> **情報を共有するために生まれた仕組み**

と言えます。

# 4. ネットワークとは

ネットワーク（Network）とは、

> **複数の機器が、共通のルールに従って情報を交換する仕組み**

です。

その共通のルールを**プロトコル（Protocol）**と呼びます。

例として、

-   IP
-   TCP
-   UDP
-   HTTP
-   HTTPS
-   DNS

などがあり、この教科書で順番に学んでいきます。

------------------------------------------------------------------------

# 5. LAN・WAN・インターネット

## LAN（Local Area Network）

LANとは、家庭・学校・会社など**限られた範囲**で利用されるネットワークです。

例 - 家庭内のWi-Fi - 会社の社内ネットワーク - 学校のコンピュータ教室

LANは比較的高速で、利用者自身や組織が管理できることが特徴です。

------------------------------------------------------------------------

## WAN（Wide Area Network）

WANは、離れた場所にあるLAN同士を接続する広域ネットワークです。

例 - 本社と支社を接続する専用線 - インターネット回線 -
通信事業者のネットワーク

WANでは通信事業者の設備を利用することが一般的です。

------------------------------------------------------------------------

## インターネットとは

インターネットは、世界中のLANやWANが相互接続された巨大なネットワークです。

インターネットそのものはサービス名ではなく、「ネットワークの集合体」を意味します。

------------------------------------------------------------------------

# 6. Webとインターネットの違い

「インターネット」と「Web」は同じ意味で使われることがありますが、厳密には異なります。

-   インターネット：通信を行うための基盤
-   Web：その基盤上で提供されるサービスの一つ

Web以外にも次のようなサービスがあります。

-   メール（SMTP / IMAP）
-   SSH（リモートログイン）
-   FTP（ファイル転送）
-   DNS（名前解決）

つまり、Webはインターネット上で動作する数あるサービスの一つです。

------------------------------------------------------------------------

# 7. クライアント・サーバモデル

現在のネットワークサービスの多くは、クライアント・サーバモデルを採用しています。

-   クライアント：サービスを利用する側
-   サーバ：サービスを提供する側

例

-   ブラウザ → クライアント
-   Webサーバ → サーバ

ブラウザはリクエストを送り、サーバはレスポンスを返します。

この考え方はWebだけでなく、メール、SSH、データベースなど多くのシステムで利用されています。

------------------------------------------------------------------------

# 8. Linuxではどのように利用されるか

Linuxでは多くのコマンドがネットワーク機能を利用しています。

  コマンド   用途
  ---------- ----------------------
  ping       疎通確認
  ssh        リモートログイン
  scp        ファイル転送
  curl       HTTP通信
  ip         ネットワーク設定確認
  ss         TCP/UDP接続確認

これらのコマンドの内部では、Socket
APIやTCP/IPプロトコルが利用されています。

代表的な確認例を見てみましょう。

```bash
ip -br address
ping -c 2 192.0.2.1
ss -tn
```

代表的な出力例（必要な部分のみ抜粋）

```text
eth0             UP             192.0.2.10/24

64 bytes from 192.0.2.1: icmp_seq=1 ttl=64 time=0.62 ms
64 bytes from 192.0.2.1: icmp_seq=2 ttl=64 time=0.55 ms

State  Recv-Q Send-Q Local Address:Port   Peer Address:Port
ESTAB  0      0      192.0.2.10:52014     198.51.100.20:443
```

確認ポイント

- `UP` はインターフェースが使用可能な状態であることを示します。
- `192.0.2.10/24` のような値が設定中のIPアドレスとプレフィックス長です。
- `ttl` は応答パケットの生存時間、`time` は往復時間の目安です。
- `ESTAB` はTCP接続が確立している状態です。

# 9. 実務ではどう役立つか

ネットワーク障害では、原因を層ごとに切り分けることが重要です。

例

-   ケーブルが抜けている
-   IPアドレス設定が間違っている
-   DNSで名前解決できない
-   Firewallで通信が遮断されている
-   Webサーバが停止している

ネットワーク全体を俯瞰できると、原因の切り分けがしやすくなります。

# 13. 解答・解説

## 問1

### 問題

LANとWANの違いを説明してください。

### 解答

LAN（Local Area
Network）は家庭・学校・会社など限られた範囲で利用されるネットワークです。

WAN（Wide Area
Network）は離れた場所にあるLAN同士を接続する広域ネットワークです。

### 解説

LANは利用者や組織が管理することが多く、高速で低遅延な通信が特徴です。

一方、WANは通信事業者の設備を利用して広域を接続します。

------------------------------------------------------------------------

## 問2

### 問題

インターネットとWebの違いを説明してください。

### 解答

インターネットは世界中のネットワークを接続する通信基盤です。

Web（World Wide
Web）は、その通信基盤上でHTTP/HTTPSを利用して提供されるサービスの一つです。

### 解説

インターネット上ではWeb以外にもメール（SMTP）、SSH、FTPなど多くのサービスが動作しています。

------------------------------------------------------------------------

## 問3

### 問題

プロトコルとは何ですか。

### 解答

プロトコルとは、機器同士が通信するための共通のルール（約束事）です。

### 解説

ネットワーク上の機器が正しく通信するためには、同じルールに従う必要があります。
TCP、IP、HTTP、DNSなどはすべてプロトコルです。

------------------------------------------------------------------------

## 問4

### 問題

クライアントとサーバの役割を説明してください。

### 解答

クライアントはサービスを利用する側、サーバはサービスを提供する側です。

### 解説

Webブラウザがクライアント、Webサーバがサーバの代表例です。

------------------------------------------------------------------------

## 問5

### 問題

Linuxでネットワーク疎通を確認する代表的なコマンドを3つ挙げてください。

### 解答例

-   ping
-   ip
-   ss

（curl、ssh、traceroute、tcpdump なども代表例です。）

### 解説

用途に応じて確認する情報が異なります。

  コマンド   主な用途
  ---------- ------------------------------
  ping       疎通確認
  ip         IPアドレス・ルーティング確認
  ss         TCP/UDP接続確認
  curl       HTTP通信確認
  ssh        リモート接続確認

# 14. 実務で考えてみよう

## ケース1：SSHでサーバへ接続できない

### 問題

SSHでサーバへ接続できません。
考えられる原因をできるだけ多く挙げてみましょう。

### 解答例

-   LANケーブルが抜けている
-   Wi-Fiが切断されている
-   IPアドレス設定が誤っている
-   デフォルトゲートウェイが誤っている
-   DNSで名前解決できない
-   Firewallで22番ポートが遮断されている
-   SSHサーバ（sshd）が停止している
-   サーバ自体の電源が入っていない

### 実務ポイント

ネットワーク障害では、最初から一つの原因を決めつけず、

**「どこまでは正常に動作しているか」**

を順番に確認することが重要です。

例えば、

1.  物理接続（ケーブル・Wi-Fi）
2.  IPアドレス
3.  ルーティング
4.  DNS
5.  TCP接続
6.  SSHサービス

というように、下位層から上位層へ切り分けることで効率よく原因を特定できます。
