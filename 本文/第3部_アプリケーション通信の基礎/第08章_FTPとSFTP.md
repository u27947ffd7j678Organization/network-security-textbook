# 第08章 FTP・SFTP

**― ファイル転送方式の違いを理解する ―**

> この章では、FTPが制御接続とデータ接続を分ける理由を中心に学びます。

------------------------------------------------------------------------

# 1. この章で学べること

- FTPが制御接続とデータ接続を分ける理由
- アクティブ・パッシブモード
- 平文FTPの問題
- SFTP・scpとの違い
- Linuxで安全にファイルを転送する方法

# 2. この章の位置付け

SSHで遠隔操作を学びました。本章ではファイル転送を目的とするFTPと、SSHを利用するSFTP・scpを比較します。

# 3. なぜこの技術が必要になったのか

大きなファイルや多数のファイルを遠隔へ送る共通手段が必要です。FTPは古くから使われますが、制御とデータを別接続にする設計や平文認証が、現代のNAT・Firewall・安全性で課題になります。

# 4. 技術の概要

**FTP（File Transfer Protocol）**は通常TCP 21番の制御接続と別のデータ接続を使います。標準FTPは認証情報と内容を暗号化しません。**SFTP（SSH File Transfer Protocol）**はSSH上で動作する別プロトコルで、通常TCP 22番を使います。FTPSはFTPをTLSで保護する方式で、SFTPとは異なります。

# 5. 詳しい仕組み

## アクティブとパッシブ

アクティブモードではサーバからクライアントへデータ接続を開始します。パッシブモードではクライアントからサーバ指定ポートへ接続するため、NATやFirewall越しで扱いやすい場合があります。それでも複数ポートの許可とアドレス通知を正しく設計する必要があります。

## SFTPとscp

SFTPは一覧取得、再開、属性操作などファイル転送用の操作を提供します。scpはSSHを利用するコピーコマンドです。現在のOpenSSHのscpは実装によりSFTPを転送方式として使います。

## 選択

新規設計では平文FTPを避け、SFTP、scp、HTTPSなど要件に合う暗号化方式を選びます。相互運用要件でFTPが必要なら、閉域・TLS・認証・ログ・最小権限を検討します。

# 6. Linuxではどうなるか

```bash
# SFTPで接続
sftp user@server.example.com

# scpでファイルを送信
scp report.txt user@server.example.com:/srv/upload/

# SSH接続の詳細を確認
sftp -v user@server.example.com
```

代表的な出力例（必要な部分のみ抜粋）

```text
$ sftp user@server.example.com
Connected to server.example.com.
sftp> put report.txt
Uploading report.txt to /home/user/report.txt
report.txt 100% 2048 1.2MB/s 00:00

$ scp report.txt user@server.example.com:/srv/upload/
report.txt 100% 2048 1.1MB/s 00:00
```

確認ポイント

- `Connected` 後、転送先パスと完了率を確認します。
- 転送後は必要に応じてサイズやハッシュ値を比較します。
- 接続先ホスト鍵の検証はSSHと同様です。

# 7. 実務ではどう使われるか

## 実務コラム：転送は成功したがアプリケーションが読めない

所有者、グループ、権限、転送先パスを確認します。

```bash
stat /srv/upload/report.txt
sha256sum report.txt /srv/upload/report.txt
```

代表的な出力例（必要な部分のみ抜粋）

```text
Access: (0640/-rw-r-----) Uid: (1001/user) Gid: (1002/upload)
abc123... report.txt
abc123... /srv/upload/report.txt
```

確認ポイント

- 実行プロセスが所有者・グループ・権限から読めるか確認します。
- ハッシュ一致は内容が同じことの確認に使えます。

# 8. FE/APではどう問われるか

FTPの制御・データ接続、アクティブ・パッシブ、FTPとSFTP・FTPSの違い、代表ポートが問われます。名前が似ていても別方式である点を整理します。

# 9. まとめ

- FTPは制御接続とデータ接続を分け、標準では暗号化しません。
- SFTPはSSH上のファイル転送プロトコルで、FTPとは別物です。
- 転送後は内容だけでなく権限と配置も確認します。

# 10. 理解度チェック

1. FTPで制御接続とデータ接続は何を担当しますか。
2. SFTPとFTPSの違いを説明してください。
3. パッシブモードがNAT環境で扱いやすいことがある理由は何ですか。

# 11. 解答・解説

## 問1

制御接続はコマンド・応答、データ接続はファイル内容や一覧を運びます。

## 問2

SFTPはSSH上の別プロトコル、FTPSはFTPをTLSで保護する方式です。

## 問3

データ接続もクライアント側から開始し、外部から内部クライアントへの新規接続を避けやすいためです。

# 12. 実務で考えてみよう

## ケース：転送途中で止まる

### 解答例

制御接続だけでなくデータ接続のポート、NAT、Firewall、パッシブモードの通知アドレスを確認します。SFTPならSSH接続、容量、権限、タイムアウトを調べます。
